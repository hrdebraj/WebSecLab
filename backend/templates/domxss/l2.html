01Recon the filter

Read the JS in Source. It applies a regex to remove <script>…</script> pairs (case-insensitive) before inserting into innerHTML. List every weakness you can think of just by reading that regex.
💡 what to look for
  • Only one tag is blacklisted. What about others?
  • Regex isn't recursive: what if tags are nested, e.g. <scr<script>ipt>?
  • Can event handlers on non-script tags still fire?
  • Whitespace, uppercase, URL encoding — does the filter see the raw URL or decoded value?

02Baseline: plain text

?q=guest — greets you. Confirm it is inserted via innerHTML (render an HTML tag like ?q=<b>x</b> to see the sink is HTML-rendering).

03Classic DOM payloads

Which DOM sinks never run <script> anyway? Right — the same rule as Level 1. So bypass is about choosing any tag that auto-fires an event handler:
  • <img> with onerror
  • <svg> with onload
  • <iframe> with onload
  • <body> with onload (only fires inside certain contexts)
Try one in ?q=. Did it fire?

04Regex recursion pitfall

Even if the filter also blacklisted onerror or img, a non-recursive replace pass can miss nested patterns. Feed it something it removes once to produce a working tag: <scri<script>pt>.... Useful to know for other labs.

05Capture the flag

As before, run alert(window.FLAG) or call revealFlag(). The flag box only appears after the payload fires.

06Burp workflow

Build the payload in Repeater or in Burp's Chromium with DOM Invader enabled. DOM Invader marks the source (URLSearchParams) and sink (innerHTML) and annotates which characters are required by the surrounding context.

07Reflection

Blacklists are a losing game. Why is an allow-list (and a library that parses HTML properly rather than regexes it) the only sensible approach?

Vulnerable script (inline)

function naiveFilter(s) {
  // Developer thought removing <script> tags was enough.
  return s.replace(/<script[^>]*>[\\s\\S]*?<\\/script>/gi, '');
}

function render() {
  var q = new URLSearchParams(location.search).get('q') || 'guest';
  document.getElementById('greet').innerHTML = 'Hello, ' + naiveFilter(q) + '!';
}
